# 大神燒肉 材料管理 ERP v3.2

這是大神燒肉材料管理 ERP 的 v3.2 版本，加入「採購紀錄編輯／作廢」功能，讓日常輸入錯誤可以在網站前端修正，不必直接進 Google Sheet 改資料。

## v3.2 新增功能

- 採購紀錄每列新增「編輯」按鈕
- 採購紀錄每列新增「作廢」按鈕
- 編輯後自動重算：金額、標準數量、標準單位價格
- 作廢不刪除資料，只標記狀態為「已作廢」
- 作廢必須填寫原因
- 記錄最後修改人 ID / 姓名
- 記錄作廢人 ID / 姓名 / 時間
- Dashboard、支出分析、價格比較自動排除已作廢資料
- 匯出 CSV 會保留狀態與作廢資訊，方便稽核

## 更新方式

### 1. 覆蓋 GitHub 前端檔案

請上傳並覆蓋：

```text
index.html
app.js
style.css
data.js
README.md
```

GitHub 操作：

```text
Repository → Add file → Upload files → Commit changes
```

### 2. 覆蓋 Apps Script 後端

打開：

```text
google-apps-script/Code.gs
```

將內容全部複製到 Apps Script 的 `Code.gs`，覆蓋舊版。

然後一定要重新部署：

```text
部署 → 管理部署作業 → 鉛筆 → 新版本 → 部署
```

只按「儲存」不會更新正式網站使用的後端。

## Google Sheet 欄位

Apps Script 會自動補上缺少欄位，但建議確認 `採購紀錄` 最右邊有：

```text
狀態
作廢原因
作廢人ID
作廢人姓名
作廢時間
```

完整欄位可看：

```text
docs/google-sheet-headers.md
```

## 測試流程

### 編輯測試

1. 登入 boss 帳號
2. 到「採購紀錄」
3. 點任一筆「編輯」
4. 修改數量或單價
5. 儲存
6. 回 Google Sheet 確認：
   - 金額已重新計算
   - 更新時間已更新
   - 最後修改人為目前登入者

### 作廢測試

1. 到「採購紀錄」
2. 點任一筆「作廢」
3. 填寫原因，例如：測試資料作廢
4. 確認作廢
5. 回 Google Sheet 確認：
   - 狀態 = 已作廢
   - 作廢原因有寫入
   - 作廢人姓名有寫入
   - 作廢時間有寫入

## 操作原則

正式資料不建議直接刪除。若輸入錯誤、重複建檔、AI 辨識錯誤，建議使用「作廢」保留痕跡，讓日後查帳有依據。
