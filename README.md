# 大神燒肉 材料管理 ERP v3.3

這是大神燒肉材料管理 ERP 的 v3.3 版本，加入「員工權限控管」。

v3.2 已完成採購紀錄編輯與作廢；v3.3 則進一步讓不同角色只能操作自己被授權的功能，避免員工誤改、誤作廢或誤用 AI 辨識。

## v3.3 新增功能

- 依「員工帳號」分頁的 `角色` 欄位決定權限
- 前端會依角色隱藏或停用不該使用的功能
- 後端 Apps Script 也會再次驗證權限，避免只靠前端限制
- 採購紀錄的「編輯」按鈕會依權限顯示
- 採購紀錄的「作廢」按鈕只開放老闆／管理員
- 採購角色只能編輯自己建立的採購紀錄
- 員工角色只能新增採購，不能編輯、作廢、AI辨識、匯出
- 唯讀角色只能查看資料，不能新增或修改
- 設定頁會顯示目前登入者的權限項目

## 角色權限表

| 角色 | 新增採購 | 編輯採購 | 作廢採購 | AI辨識 | 支出分析 | 匯出CSV | 材料管理 |
|---|---|---|---|---|---|---|---|
| 老闆 | 可 | 全部 | 可 | 可 | 可 | 可 | 可 |
| 管理員 | 可 | 全部 | 可 | 可 | 可 | 可 | 可 |
| 採購 | 可 | 只可編輯自己建立 | 不可 | 可 | 可 | 可 | 不可 |
| 員工 | 可 | 不可 | 不可 | 不可 | 不可 | 不可 | 不可 |
| 唯讀 | 不可 | 不可 | 不可 | 不可 | 可 | 可 | 不可 |

## 更新方式

### 1. 覆蓋 GitHub 前端檔案

請上傳並覆蓋：

```text
index.html
app.js
style.css
data.js
README.md
```

GitHub 操作：

```text
Repository → Add file → Upload files → Commit changes
```

### 2. 覆蓋 Apps Script 後端

打開：

```text
google-apps-script/Code.gs
```

將內容全部複製到 Apps Script 的 `Code.gs`，覆蓋舊版。

然後一定要重新部署：

```text
部署 → 管理部署作業 → 鉛筆 → 新版本 → 部署
```

只按「儲存」不會更新正式網站使用的後端。

## Google Sheet 設定

員工權限直接使用 `員工帳號` 分頁的 `角色` 欄位。

建議角色只能填下列其中一種：

```text
老闆
管理員
採購
員工
唯讀
```

範例：

```text
EMP-0001	阿狼	boss	你的密碼			老闆	是				系統管理者
EMP-0002	小光	staff01	123456			員工	是				一般員工
EMP-0003	採購測試	buyer01	123456			採購	是				採購人員
EMP-0004	唯讀測試	viewer01	123456			唯讀	是				只看資料
```

## 測試流程

### 老闆測試

1. 使用 `boss` 登入
2. 採購紀錄應看到「編輯」與「作廢」
3. 可新增、編輯、作廢、AI辨識、匯出CSV

### 採購測試

1. 新增一個角色為 `採購` 的帳號
2. 使用該帳號新增一筆採購
3. 該筆資料應可編輯
4. 其他人建立的資料應不可編輯
5. 不應看到作廢按鈕

### 員工測試

1. 新增一個角色為 `員工` 的帳號
2. 可新增採購
3. 不可編輯、作廢、AI辨識、匯出CSV

### 唯讀測試

1. 新增一個角色為 `唯讀` 的帳號
2. 不可新增採購
3. 不可編輯或作廢
4. 可查看資料與分析

## 安全原則

v3.3 不是只在畫面上隱藏按鈕。Apps Script 後端也會檢查角色權限，因此即使有人嘗試直接呼叫 API，也會被後端拒絕。
